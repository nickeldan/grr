Determines if a string contains a substring which matches the regex.

The function will stop processing characters when either it exhausts the specified length, a newline is encountered, or a non-printable character is encountered with tolerateNonprintables set to false. If tolerateNonprintables is true, then the function will skip over such characters and treat them as beginning and end of lines. For example, if the regex is "^a+" and the string is "aa\x00aaad", then both sequence of "a"'s will be considered matches. However, they will be considered separate matches and so *end-*start will be 3.

Parameters
nfaThe Grr regex object.
stringThe string (does not have to be null-terminated).
lenThe length of the string.
startA pointer which will, if not NULL, point to the index of the beginning of the longest match if one was found.
endA pointer which will, if not NULL, point to the index of the character after the end of the longest match if one was found.
cursorA pointer which will, if not NULL, point to the index of the character where the function stopped searching.
tolerateNonprintablesIndicates whether or not the function should exit upon encountering a non-printable character.
Returns
GRR_RET_OK if a substring match was found. GRR_RET_NOT_FOUND if no substring match was found. GRR_RET_BAD_DATA if the string contained a non-printable character and tolerateNonprintables was set to false.
